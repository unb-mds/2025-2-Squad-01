Arquitetura do Sistema

Visão Geral

O projeto CoOps utiliza uma arquitetura moderna e modular, baseada em tecnologias web e integração com APIs do GitHub para análise de métricas de colaboração em repositórios e organizações.

ADR 01: Escolha das Tecnologias para Desenvolvimento do Projeto ✅ Aceito

Contexto

Nas discussões iniciais do projeto foi debatido sobre quais tecnologias utilizar.

Decisão

As tecnologias definidas foram:

  • Python
  • React
  • Biblioteca D3 (JavaScript)
  • API GitHub
  • GitHub Actions
  • Firebase
  • Arquivos JSON para armazenamento e transferência dos dados

Detalhamento das Tecnologias

Python

Utilizado para a criação dos scripts que irão realizar o tratamento dos dados extraídos da API do GitHub.

React

Utilizado em conjunto com a biblioteca D3 para a criação da interface frontend, que consumirá os arquivos JSON e exibirá os gráficos com as métricas do GitHub.

Biblioteca D3 (JavaScript)

Biblioteca de código aberto para visualização de dados. Permite a construção de gráficos personalizados. Será utilizada junto com o React para consumir os dados dos arquivos JSON e criar as páginas de visualização gráfica das métricas do GitHub.

API GitHub

Utilizada para extrair os dados de repositórios e organizações do GitHub (commits, issues, datas de abertura e fechamento de issues, e pull requests).

GitHub Actions

Garante a atualização permanente dos arquivos JSON e, consequentemente, uma análise sempre atualizada das métricas. Essa integração com os arquivos JSON dispensa a necessidade de uma aplicação backend e simplifica a implementação do projeto.

Firebase

Utilizado para realizar a hospedagem do projeto, proporcionando uma solução escalável e de baixo custo.

Consequências e Benefícios

As tecnologias escolhidas proporcionam:

  • Maior tempo de permanência do projeto: Utilizando tecnologias estáveis e amplamente suportadas.
  • Análises atualizadas: Com base nas métricas do GitHub através de automação via GitHub Actions.
  • Redução de custos com hospedagem: Utilizando Firebase para hospedagem estática.
  • Simplicidade arquitetural: Dispensando a necessidade de um backend complexo através do uso de arquivos JSON e GitHub Actions.

Fluxo de Dados

  1. Extração: Scripts Python utilizam a API do GitHub para extrair dados de repositórios e organizações.
  2. Processamento: Os dados são processados e transformados em arquivos JSON estruturados.
  3. Atualização: GitHub Actions executa periodicamente os scripts para manter os dados atualizados.
  4. Visualização: O frontend React consome os arquivos JSON e utiliza D3.js para gerar visualizações interativas.
  5. Hospedagem: O projeto é servido através do Firebase para usuários finais.

Vantagens da Arquitetura Escolhida

  • Escalabilidade: Cada componente pode ser escalado independentemente.
  • Manutenibilidade: Separação clara de responsabilidades entre camadas.
  • Performance: Dados pré-processados em JSON resultam em carregamento rápido.
  • Custo-benefício: Utilização de serviços gratuitos ou de baixo custo.
  • Automatização: Processo completamente automatizado de atualização de dados.
